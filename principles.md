# Principles — Cross-Asset Voice + Philosophy

This file is the cross-channel substrate. Channel-specific rules live in `wiki/voices/*` and `wiki/channels/*`. This file holds what's true *everywhere*.

## Voice principles (apply to every channel)

1. **Stories, not summaries.** Never react to someone else's article — write a story.
2. **Real numbers.** Time anchors, dollar/shekel amounts, dates. No vague "recently" / "many."
3. **Binary or triple closer.** End with a forced choice or three sharp options. Never trail off.
4. **Forbidden phrases (kill list, expanded per channel):** see `wiki/voices/*`.
5. **Show the thinking, not the conclusion.** Anti-LinkedIn-thought-leader.
6. **Anonymize clients.** "לקוח" / "the client" / "a founder I met." Never name unless permission.

## Content philosophy

- **The builders win.** Every asset implicitly answers: "Can someone DO something after reading this?"
- **Career relevance** (LI) / **Practical agency** (FB) — every post passes the use test.
- **Boutique not bazaar.** Pricing, audience, channel — quality over volume.

## Cross-asset patterns

- Hebrew: minimize hyphens (no "ב-2000", write "ב2000")
- English: never name living tools/brands unless used as case study
- All: never invent personal scenes about Yuval (parenting, kitchen, etc.)
- Hebrew: "סוכן" not "סוכנית" — סוכנית is not a word

## Cross-channel kill list (excerpts — full lists per voice)

- "הרס לי את הראש", "שבר לי את הראש" — forced slang
- "אחי" — never (forced/unnatural)
- "Microsoft told designers" — overused angle
- "להזיע בטוסיק" — vulnerability phrases never recycled
