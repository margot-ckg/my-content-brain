# Writing small

**Definition:** Default to fewer words. Every word must earn its place. Customers come to accomplish tasks, not read.

**When to use:**
- Always — this is the base discipline, not a special case.

**Core moves:**

### 1. Front-load the key message
Put what the customer needs to know in the first 4–5 words. The rest is supporting detail.
- "Let's get your W-2 / To get started, we'll need a digital copy…" ✓
- "OK, if you have a pdf copy of your W-2, we can use it to save time" ✗

### 2. Make it scannable
Customers skim. Use hierarchy, bullets, and visual space to guide them to what matters.

### 3. Headlines: tight and specific
Headlines should quickly convey the gist. Cut ruthlessly.
- "Get expert help with Full Service" ✓ (11 fewer words than the alternative)

### 4. Use progressive disclosure
Don't dump all information at once. Layer it. Especially important for:
- Long forms / multi-step flows
- Neurodivergent users (ADHD, anxiety)
- Mobile contexts

Progressive disclosure methods:
- Links to detail pages
- Accordions (use thoughtfully — hiding info has a cost)
- Conditional fields (show follow-up fields based on selection)
- Step-throughs with logically clustered questions

### 5. Challenge every word
Look word by word: does this word add meaning, or is it filler? If you can cut it without losing meaning, cut it. Only add back if meaning suffers.

**Checklist before shipping copy:**
- [ ] Is the key message in the first few words?
- [ ] Can any word be removed without losing meaning?
- [ ] Would a bullet list make this clearer than a paragraph?
- [ ] Is the CTA doing enough informational work (especially in modals)?
