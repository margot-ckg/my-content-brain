# Empty-state invitation

**Definition:** Open an empty state with a one-line invitation to take the primary action. No "Nothing here yet". No apology for the emptiness. Just the next step.

**When to use:**
- Empty states for a feature the user is meant to populate (no projects yet, no contacts yet, no transactions yet).
- Onboarding screens before the user has done anything.

**When NOT to use:**
- States where emptiness is the desired outcome (inbox zero, no errors, no debts).
- When the user has already engaged but the data isn't loading (use loading or error states instead).
